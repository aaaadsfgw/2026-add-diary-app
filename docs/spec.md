# Diary App 仕様書

## 1. プロジェクト概要

Diary App は、スマートフォン上で日記を作成・閲覧・編集・検索できる日記アプリである。  
React Native / Expo を用いて開発し、日記データは Firebase または Supabase などの外部サーバーに保存する。

## 2. 目的

ユーザーが日々の出来事や気分を簡単に記録し、後から一覧・詳細・検索によって振り返れるようにすることを目的とする。

## 3. 対象ユーザー

日常的にスマートフォンで日記を記録したいユーザーを対象とする。  
短時間で入力でき、あとから過去の日記を探しやすいことを重視する。

## 4. 必須要件

- 外部のサーバーに日記データを保存する
- 日記一覧、日記作成、日記詳細、日記編集、日記検索画面を作成する
- 日記一覧では無限スクロールを使う
- 日記検索はホーム画面に内包する、または専用画面として作成する
- 日記にはアイコン、タイトル、本文、日付、画像を入力できるようにする
- 画像は任意入力とし、それ以外は必須入力とする

## 5. 機能要件

### 5.1 日記一覧機能

- 保存された日記を新しい順に一覧表示する
- 一覧にはアイコン、タイトル、日付、本文の一部を表示する
- スクロールに応じて追加の日記を読み込む
- 各日記を選択すると詳細画面へ移動する

### 5.2 日記作成機能

- アイコン、タイトル、本文、日付、画像を入力できる
- 画像以外の項目は必須入力とする
- 入力内容を外部サーバーに保存する
- 保存後は日記一覧または詳細画面へ遷移する

### 5.3 日記詳細機能

- 選択した日記の全内容を表示する
- アイコン、タイトル、本文、日付、画像を表示する
- 編集画面へ移動できる

### 5.4 日記編集機能

- 既存の日記内容を編集できる
- 画像の追加・変更・削除を行える
- 保存時に外部サーバー上のデータを更新する

### 5.5 日記検索機能

- タイトルや本文を対象に日記を検索できる
- 検索結果を一覧で表示する
- 検索機能はホーム画面内、または専用画面に実装する

## 6. 非機能要件

### 6.1 操作性

- スマートフォンで操作しやすいUIにする
- 入力項目は分かりやすく配置する
- 日記作成までの操作数をできるだけ少なくする

### 6.2 パフォーマンス

- 日記一覧は無限スクロールを使い、一度に大量のデータを読み込まない
- 画像を扱う場合でも画面表示が重くなりすぎないようにする

### 6.3 保守性

- 画面、データ処理、外部サーバー処理を分けて実装する
- 機能ごとにファイルを整理し、後から変更しやすい構成にする

### 6.4 セキュリティ

- APIキーなどの秘密情報をコード内に直接書かない
- 外部サーバーのアクセス権限を適切に設定する

## 7. 技術スタック

- React Native
- Expo
- TypeScript
- Firebase または Supabase
- pnpm
- Expo Go

## 8. データ項目

| 項目 | 型 | 必須 | 説明 |
|---|---|---|---|
| id | string | 必須 | 日記を識別するID |
| icon | string | 必須 | 日記に表示するアイコン |
| title | string | 必須 | 日記のタイトル |
| body | string | 必須 | 日記の本文 |
| date | string | 必須 | 日記の日付 |
| imageUrl | string | 任意 | 画像のURL |
| createdAt | string | 必須 | 作成日時 |
| updatedAt | string | 必須 | 更新日時 |

## 9. 画面構成

- ホーム画面
  - 日記一覧
  - 検索フォーム
  - 日記作成画面への導線
- 日記作成画面
- 日記詳細画面
- 日記編集画面

## 10. 開発方針

まずは日記の一覧表示、作成、詳細表示を優先して実装する。  
その後、編集、検索、画像アップロード、外部サーバー連携、無限スクロールの順に拡張していく。  
機能を一つ追加するごとに、Expo Goで動作確認を行う。
